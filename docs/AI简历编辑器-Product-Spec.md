# AI 简历编辑器 Product Spec

## 产品说明

AI 简历编辑器是一个面向通用求职者的在线简历制作与修改工具。用户从空白简历开始，通过左侧聊天栏用自然语言告诉 AI 自己的经历、目标岗位和修改意图，AI 负责生成、改写、重排和美化简历内容。右侧实时显示 A4 HTML 简历预览，用户可以查看版本历史、回滚到任意历史版本，并导出 HTML 或 PDF 文件。

产品的核心价值是把「写简历、调排版、改内容、导出文件」压缩到一个聊天式工作流里，让求职者把精力放在经历表达和岗位匹配上。

## 功能需求

- 创建空白简历：用户首次进入产品后创建一份空白简历，系统生成默认 A4 页面、基础模块和默认样式。
- 通过聊天修改简历：用户在左侧聊天栏输入自然语言指令，AI 根据指令修改结构化简历数据、样式配置和 HTML 预览。
- 自由编辑内容与版式：AI 支持修改文案、模块顺序、模块增删、标题命名、字体、字号、间距、颜色、分栏、强调样式和整体视觉风格。
- 实时预览 A4 简历：右侧以 A4 纸张比例渲染 HTML 简历，支持缩放、滚动和分页边界提示。
- 保存版本历史：每次 AI 成功修改后生成一个版本记录，记录用户指令、简历 JSON、样式配置、HTML 快照和生成时间。
- 回滚历史版本：用户可以打开版本面板，查看历史修改记录，并恢复到指定版本。
- 导出 HTML：用户可以导出当前简历的完整 HTML 文件，文件包含样式、内容和必要资源引用。
- 导出 PDF：用户可以导出当前 A4 简历为 PDF，保持右侧预览的页面尺寸、分页和样式。
- 提供 AI 修改确认：AI 修改前展示本次计划改动摘要，用户确认后应用到右侧预览。
- 处理异常状态：AI 修改失败、PDF 生成失败、版本恢复失败时，系统展示明确错误信息，并保留当前可用版本。

## UI 布局

### 整体布局

产品采用左右两栏工作台布局，页面高度占满浏览器视口。

- 左侧聊天栏占 32% 宽度，最小宽度 360px，最大宽度 520px。
- 右侧预览区占剩余宽度，用于展示 A4 简历和顶部工具栏。
- 两栏之间提供可拖拽分隔线，用户可以调整聊天栏宽度。
- 页面顶部保持轻量工具栏，显示产品名称、当前简历标题、保存状态和导出入口。

### 左侧 - 聊天栏

左侧用于和 AI 对话，包含三个区域：

- 顶部状态区：显示当前简历名称、目标岗位输入入口、最近一次保存时间。
- 中部消息区：展示用户消息、AI 回复、AI 改动摘要和确认控件。
- 底部输入区：多行输入框加发送按钮，支持快捷指令按钮。

聊天栏控件规范：

- 多行输入框高度 96px，可随内容增长到 180px。
- 发送按钮位于输入框右下角，主按钮样式。
- 快捷指令位于输入框上方，横向排列，默认提供「生成第一版」「优化表达」「匹配岗位」「精简一页」「调整版式」。
- AI 改动摘要卡片展示本次将修改的模块、样式和影响范围，底部放置「应用修改」和「重新生成」按钮。
- 版本入口放在聊天栏顶部右侧，点击后打开版本历史抽屉。

### 右侧 - A4 预览区

右侧用于展示和导出简历，包含工具栏、画布区和页面预览。

- 顶部工具栏：放置缩放控件、页面适配控件、导出 HTML 按钮、导出 PDF 按钮。
- 画布区：居中展示 A4 页面，背景使用浅灰工作区。
- A4 页面：固定 210mm × 297mm 比例，白色背景，细边框和轻微阴影。
- 页面内容：由 HTML 模板根据 `resume.json` 和 `style.json` 渲染生成。
- 缩放选项：50%、75%、100%、适合宽度、适合页面。
- 分页提示：内容超过一页时显示第二页 A4 页面，页间距固定。

### 版本历史抽屉

版本历史从右侧或左侧覆盖层打开，宽度 360px。

- 每个版本项显示生成时间、用户指令摘要、AI 修改摘要和操作按钮。
- 当前版本显示选中状态。
- 用户点击「恢复」后，右侧预览立即切换到该版本。
- 版本列表按时间倒序排列。

## 用户使用流程

### 首次创建简历

1. 用户进入产品，点击「创建空白简历」。
2. 系统生成默认简历标题、空白 `resume.json`、默认 `style.json` 和基础 HTML 模板。
3. 右侧显示空白 A4 简历，左侧出现 AI 引导消息。
4. 用户输入个人信息、教育经历、工作经历、项目经历或目标岗位。
5. AI 生成第一版简历内容，并展示改动摘要。
6. 用户点击「应用修改」。
7. 系统更新右侧 A4 预览，并保存版本记录。

### 聊天修改简历

1. 用户在聊天栏输入修改指令，例如「把项目经历写得更像后端开发岗位」。
2. AI 读取当前 `resume.json`、`style.json`、HTML 预览摘要和版本上下文。
3. AI 返回修改计划，包括要修改的模块、主要文案变化和版式变化。
4. 用户确认应用。
5. 系统执行结构化 patch，更新简历数据和样式配置。
6. 右侧 HTML 预览重新渲染。
7. 系统生成新版本记录。

### 回滚版本

1. 用户打开版本历史抽屉。
2. 用户查看历史版本列表和每次修改摘要。
3. 用户点击目标版本的「恢复」按钮。
4. 系统将当前 `resume.json`、`style.json` 和 HTML 快照恢复到目标版本。
5. 右侧预览刷新，并生成一条「恢复版本」记录。

### 导出 HTML

1. 用户点击右上角「导出 HTML」。
2. 系统基于当前 `resume.json`、`style.json` 和模板生成完整 HTML。
3. 系统下载 `.html` 文件，文件名使用简历标题和日期。

### 导出 PDF

1. 用户点击右上角「导出 PDF」。
2. 系统调用浏览器打印或服务端渲染能力生成 PDF。
3. PDF 使用 A4 尺寸、打印 CSS 和当前页面样式。
4. 系统下载 `.pdf` 文件，文件名使用简历标题和日期。

## AI 能力需求

- 结构化简历生成：根据用户输入生成 `resume.json` 初稿。
- 内容改写：根据目标岗位优化经历表达、动词、指标和成果描述。
- 版式调整：根据用户指令修改 `style.json`，控制字体、间距、颜色、模块顺序和分栏。
- HTML 渲染辅助：根据模板规则生成可预览、可导出的 HTML。
- 修改计划生成：每次应用前输出改动摘要，方便用户确认。
- 版本摘要生成：为每次修改生成简短版本说明。
- 岗位匹配优化：根据目标岗位提取关键词，并建议简历内容强化方向。

## 应用场景

- 应届生第一次写简历：用户输入学校、专业、项目和实习经历，AI 生成第一版完整简历，并把内容压缩到一页。
- 转行求职者重写经历：用户告诉 AI 目标岗位，AI 将过往经历改写成更匹配岗位要求的表达。
- 有经验求职者快速定制版本：用户针对不同岗位输入 JD 重点，AI 调整简历关键词、模块顺序和项目描述。
- 求职前快速润色：用户把已有经历补进空白简历，AI 优化语气、量化成果和排版密度。
- 简历顾问辅助交付：顾问通过聊天快速生成多个版本，并导出 HTML/PDF 给客户确认。

## 第一版验收标准

- 用户可以创建一份空白简历。
- 用户可以通过聊天让 AI 生成和修改简历。
- 每次 AI 修改都能更新右侧 A4 HTML 预览。
- 用户可以查看版本历史并恢复指定版本。
- 用户可以导出当前简历为 HTML 文件。
- 用户可以导出当前简历为 PDF 文件。
- 当前页面刷新后可以恢复最近一次编辑状态。
