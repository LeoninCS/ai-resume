# AI 简历编辑器 - UI/UX 交互规格

> 版本：0.1  
> 日期：2026-05-19  
> 关联 PRD：docs/AI简历编辑器-Product-Spec.md

## 0. 快速摘要

- 产品一句话：面向通用求职者的 AI 简历制作与修改工具，用户通过聊天让 AI 生成、改写和排版简历，并导出 HTML/PDF。
- MVP 核心任务：用户从空白简历开始，通过聊天生成简历，查看 A4 预览，回滚版本，导出文件。
- MVP 页面：简历工作台 / 版本历史抽屉 / 导出确认弹窗。
- 设备优先级：电脑网页为主，平板可用，手机提供基础查看和简单编辑。
- 技术栈：`html-tailwind`（Tailwind CSS）。
- 风格关键词：克制、清爽、可信、工具感、低干扰。
- 本文覆盖范围：导航与布局、页面结构、组件规范、交互状态、文案、颜色、字体、间距。

## 0.1 使用方式

- 这份文档是 UI 的唯一真相，颜色、字号、间距、圆角和组件都以这里为准。
- 后续实现页面时，先使用 tokens（设计变量：全站统一的颜色、字号、间距等）和组件规范组合页面。
- 需要新增组件或 variant（同一组件的不同样式：比如主按钮/次按钮）时，先补充第 5 节组件规范。

## 1. 范围与原则

### 1.1 本期范围

- 简历工作台：左侧聊天栏，右侧 A4 HTML 预览。
- 空白创建：首次进入生成一份空白简历。
- 聊天修改：用户输入自然语言，AI 返回改动摘要，用户确认后应用。
- 版本回滚：查看版本列表，恢复指定版本。
- 导出：导出 HTML 和 PDF。
- 本地恢复：刷新后恢复最近一次编辑状态。

### 1.2 范围边界

- 模板库、多端账号同步、上传 PDF/Word 解析、在线支付、团队协作、分享链接放入后续版本。
- MVP 保留一个默认简历模板，样式自由修改由 AI 修改 `style.json` 完成。

### 1.3 设计原则

1. 工作台始终围绕「聊天指令 → 改动确认 → A4 预览」这个闭环组织。
2. 每个区域只承载一个主要任务：左侧负责沟通，右侧负责查看，弹窗/抽屉负责辅助操作。
3. 主操作保持清晰，同一状态下只突出一个最重要按钮。
4. A4 预览优先保持稳定，聊天输入、弹窗和抽屉的变化不能挤压纸张内容。
5. 用户输入和当前版本始终可保留，失败状态给出可继续操作的路径。

## 2. 页面结构与导航

### 2.1 导航模型

采用单页工作台模型。用户进入后直接看到左侧聊天和右侧简历，不经过多层导航。版本历史和导出流程通过抽屉/弹窗完成。

### 2.2 页面清单

| 页面/层级 | 目的 | 入口 | 路由/标识 | 备注 |
| --- | --- | --- | --- | --- |
| 简历工作台 | 完成创建、聊天修改、预览和导出 | 默认入口 | `/workspace` | MVP 主页面 |
| 版本历史抽屉 | 查看修改记录并恢复版本 | 工作台顶部「历史」按钮 | `version-drawer` | 覆盖在工作台上 |
| 导出确认弹窗 | 选择导出 HTML/PDF 并反馈结果 | 右上角导出按钮 | `export-dialog` | 居中弹窗 |
| 改动确认卡片 | 展示 AI 修改计划并等待确认 | 聊天消息流中 | `change-card` | 聊天内组件 |

### 2.3 全局布局骨架

- 顶部栏：左侧显示产品名和简历标题，中间显示保存状态，右侧显示「历史」「导出 HTML」「导出 PDF」。
- 左侧聊天栏：顶部为简历上下文，中部为消息流，底部为快捷指令和输入框。
- 右侧预览区：顶部为预览工具栏，中部为 A4 画布，底部可显示页数和缩放状态。
- 抽屉：从右侧滑入，用于版本历史。
- 弹窗：居中显示，用于导出确认、错误详情和恢复确认。

### 2.4 布局规则

- 页面宽度：占满浏览器视口。
- 页面高度：`100vh`，顶部栏固定高度 56px，下方工作区占满剩余高度。
- 左侧聊天栏：宽度 32%，最小 360px，最大 520px。
- 分隔线：宽度 6px，悬浮时显示拖拽提示。
- 右侧预览区：占剩余宽度，画布居中。
- 页面内边距：顶部栏左右 16px，聊天栏 16px，预览工具栏 12px 16px。
- 信息密度：工具区偏紧凑，A4 简历内容遵循真实打印密度。
- 对齐原则：工具栏水平居中对齐，聊天消息左对齐，A4 页面居中对齐。

## 3. 核心流程与关键状态

### 3.1 核心流程：从空白到导出

1. 用户进入工作台。
2. 系统创建空白简历，右侧显示 A4 空白模板，左侧显示 AI 引导消息。
3. 用户在聊天框输入经历、目标岗位或直接点击「生成第一版」。
4. AI 生成改动摘要卡片，列出内容变化和版式变化。
5. 用户点击「应用修改」。
6. 系统更新 `resume.json` 和 `style.json`，右侧 A4 预览刷新。
7. 系统保存版本记录，并在顶部栏显示「已保存」。
8. 用户继续聊天修改，或点击导出 HTML/PDF。

### 3.2 聊天修改流程

1. 用户输入指令，例如「把项目经历改得更适合后端开发岗位」。
2. 输入框进入发送中状态，消息流新增用户消息。
3. AI 回复「改动摘要」，包括修改模块、内容方向、版式影响和风险提示。
4. 用户点击「应用修改」后，卡片进入应用中状态。
5. 右侧预览显示局部刷新遮罩，完成后刷新页面内容。
6. 消息流新增成功提示，版本历史新增一条记录。

### 3.3 版本回滚流程

1. 用户点击顶部栏「历史」。
2. 右侧打开版本历史抽屉。
3. 用户点击某个版本项，右侧预览进入只读预览状态。
4. 用户点击「恢复此版本」。
5. 系统弹出确认框，显示恢复后的影响。
6. 用户确认后恢复数据，工作台回到正常编辑状态。

### 3.4 导出流程

1. 用户点击「导出 HTML」或「导出 PDF」。
2. 系统打开导出确认弹窗，显示文件名、格式和页面数量。
3. 用户点击「开始导出」。
4. 按钮进入加载态，弹窗显示进度说明。
5. 成功后下载文件，并显示 toast。
6. 失败时保留弹窗，显示错误原因和重试按钮。

### 3.5 关键状态规则

- 加载中：区域内显示骨架屏或轻量遮罩，按钮内部显示小转圈并保持原尺寸。
- 空状态：给出下一步动作，例如「输入经历生成第一版」。
- 出错：保留用户输入和当前简历，显示错误原因、重试按钮和回到上一版本入口。
- 成功：右下角 toast 显示 3 秒，文案短句清晰。

## 4. 设计系统（Tokens：设计变量）

### 4.1 风格方向

- 风格描述：清爽可信的工具型界面，界面本身退后，简历内容站到前面。
- 关键词：克制、清爽、专业、稳定、低干扰。
- 组件气质：圆角适中，边框清晰，阴影轻，色彩集中在关键操作。
- 视觉约束：少用大面积装饰、强渐变、厚重阴影、复杂背景纹理。

### 4.2 颜色 tokens

| Token | Light | 用在哪里 | 使用约束 |
| --- | --- | --- | --- |
| `--color-brand` | `#2563EB` | 主按钮、强调链接、焦点环 | 正文大段文字使用 `--color-text` |
| `--color-brand-hover` | `#1D4ED8` | 主按钮悬浮 | 仅用于交互态 |
| `--color-bg` | `#F8FAFC` | 工作台背景、画布背景 | A4 页面使用白色 |
| `--color-surface` | `#FFFFFF` | 面板、聊天栏、弹窗、抽屉 | 页面背景使用 `--color-bg` |
| `--color-text` | `#0F172A` | 正文和主要标题 | 禁用态使用 muted |
| `--color-muted` | `#64748B` | 辅助文字、时间、说明 | 关键提示使用 text |
| `--color-border` | `#E2E8F0` | 边框、分割线 | 主按钮边框跟随 brand |
| `--color-success` | `#16A34A` | 保存成功、导出成功 | 主操作仍使用 brand |
| `--color-warn` | `#D97706` | 导出耗时、恢复提示 | 只用于警示状态 |
| `--color-danger` | `#DC2626` | 失败、删除、危险确认 | 普通提示使用 muted |
| `--color-paper` | `#FFFFFF` | A4 简历纸张 | 工作台背景使用 bg |
| `--color-preview-bg` | `#E5E7EB` | A4 外围画布 | 面板背景使用 surface |

### 4.3 交互状态色

- Hover：背景色加深或填充浅蓝底，文字保持可读。
- Active：按钮颜色加深，按下态维持 120ms。
- Focus：使用 `--color-brand` 2px 外环。
- Disabled：透明度 50%，鼠标样式为不可点，保留按钮尺寸。

### 4.4 字体与排版 tokens

- 界面字体：Inter、系统无衬线字体。
- 简历字体：Arial、Times New Roman、系统宋体可由 `style.json` 控制。
- 字重策略：页面标题 600，组件标题 600，正文 400，辅助信息 400。

| Token | 字号 | 行高 | 字重 | 用在什么地方 |
| --- | ---: | ---: | ---: | --- |
| `--text-page-title` | 16px | 24px | 600 | 顶部栏标题 |
| `--text-section-title` | 14px | 20px | 600 | 面板小标题 |
| `--text-body` | 14px | 22px | 400 | 聊天消息、表单内容 |
| `--text-small` | 12px | 18px | 400 | 时间、状态、说明 |
| `--text-button` | 14px | 20px | 500 | 按钮 |
| `--text-resume-base` | 10.5pt | 1.45 | 400 | A4 简历正文 |

### 4.5 间距 tokens

| Token | px | 用在什么地方 |
| --- | ---: | --- |
| `--space-1` | 4 | 图标与文字间距 |
| `--space-2` | 8 | 小组件内部间距 |
| `--space-3` | 12 | 表单项、工具栏按钮间距 |
| `--space-4` | 16 | 面板内边距 |
| `--space-5` | 20 | 聊天消息组间距 |
| `--space-6` | 24 | 页面区块间距 |
| `--space-8` | 32 | 大区块间距 |

### 4.6 圆角、阴影、边框 tokens

### 5.1 组件清单

| 组件 | variant | 尺寸 | 状态 | 备注 |
| --- | --- | --- | --- | --- |
| Button | primary / secondary / ghost / danger | sm / md | 默认 / 悬浮 / 禁用 / 加载中 | 每个操作区只突出一个 primary |
| Textarea | default | md / lg | 默认 / 聚焦 / 错误 / 禁用 | 聊天输入使用 lg |
| MessageBubble | user / ai / system | md | 发送中 / 完成 / 失败 | 聊天消息 |
| ChangeSummaryCard | pending / applying / applied / failed | md | 等待确认 / 应用中 / 成功 / 失败 | AI 改动确认 |
| ResumePaper | page / readonly / refreshing | A4 | 正常 / 刷新中 / 溢出 | 右侧预览 |
| ToolbarButton | icon / icon-text | sm | 默认 / 悬浮 / 选中 / 禁用 | 预览工具栏 |
| VersionItem | current / preview / normal | md | 默认 / 选中 / 恢复中 | 版本历史 |
| Toast | success / warning / error | md | 进入 / 停留 / 离开 | 全局反馈 |
| Dialog | confirm / export / error | md | 打开 / 加载 / 完成 | 弹窗 |

### 5.2 Button

- primary：用于「应用修改」「开始导出」「恢复此版本」等当前区域主操作。
- secondary：用于「重新生成」「取消」「查看详情」。
- ghost：用于工具栏图标按钮、关闭按钮、低权重操作。
- danger：用于清空、删除版本等危险操作。
- 文案规则：动词开头，优先 2-6 个字，例如「应用修改」「重新生成」「开始导出」。

Tailwind 配方：

```txt
primary: bg-brand text-white hover:bg-brand-hover
secondary: border border-border bg-surface text-text hover:bg-bg
ghost: text-muted hover:bg-bg hover:text-text
danger: bg-danger text-white hover:opacity-90
loading: relative disabled:opacity-80
```

### 5.3 聊天输入框

- 位置：左侧聊天栏底部固定。
- 高度：默认 96px，最大 180px。
- 支持换行：Enter 发送，Shift+Enter 换行。
- 发送中：发送按钮变加载态，输入框保持可见。
- 错误：底部显示一行错误说明，用户原输入保留。

快捷指令：

- 「生成第一版」
- 「优化表达」
- 「匹配岗位」
- 「精简一页」
- 「调整版式」

### 5.4 消息气泡

- 用户消息：右侧对齐，浅蓝背景，最大宽度 82%。
- AI 消息：左侧对齐，白色背景，边框，最大宽度 92%。
- 系统消息：居中或左侧小字，用于保存、恢复、导出结果。
- 每条消息显示时间，时间使用 `--text-small` 和 `--color-muted`。

### 5.5 改动摘要卡片

卡片内容顺序：

1. 标题：「本次修改计划」
2. 模块列表：例如「项目经历」「技能」「版式」
3. 具体变化：最多 5 条，每条一句话。
4. 影响提示：例如「预计仍保持 1 页」。
5. 操作区：「应用修改」primary，「重新生成」secondary。

状态：

- pending：等待用户确认。
- applying：按钮加载，右侧预览显示刷新遮罩。
- applied：卡片显示绿色小标「已应用」。
- failed：卡片显示错误原因和「重试」按钮。

### 5.6 A4 预览纸张

- 纸张比例：210mm × 297mm。
- 屏幕显示：使用等比缩放，默认「适合页面」。
- 纸张背景：纯白。
- 纸张阴影：`--shadow-sm`。
- 页面间距：多页时页面间距 24px。
- 溢出提示：内容超出当前页时，在页底显示浅橙提示条「内容已进入下一页」。

### 5.7 版本项

内容：

- 时间：`2026-05-19 10:32`
- 用户指令摘要：一行，超出省略。
- AI 修改摘要：两行，超出省略。
- 状态标识：当前版本、预览中、已恢复。
- 操作：「预览」「恢复」。

交互：

- 点击版本项：右侧进入版本预览。
- 点击恢复：打开确认弹窗。
- 当前版本项固定显示在列表顶部。

### 5.8 Toast

位置：右下角。

文案：

- 保存成功：「已保存」
- 应用成功：「修改已应用」
- 导出成功：「文件已下载」
- 导出失败：「导出失败，请重试」
- 恢复成功：「已恢复到所选版本」

持续时间：成功 3 秒，失败 5 秒。

## 6. 页面规格

### 6.1 页面：简历工作台

- 目的：让用户通过聊天完成简历创建、修改、预览和导出。
- 入口：打开产品默认进入。
- 主操作：聊天栏中的「发送」或改动摘要卡片中的「应用修改」。

页面骨架：

- 顶部栏：
  - 左侧：产品名「AI 简历编辑器」、简历标题输入。
  - 中间：保存状态，如「已保存」「保存中」「保存失败」。
  - 右侧：「历史」「导出 HTML」「导出 PDF」。
- 左侧聊天栏：
  - 顶部：目标岗位输入，最近保存时间。
  - 中部：消息列表。
  - 底部：快捷指令、聊天输入框、发送按钮。
- 中间分隔线：
  - 支持左右拖拽。
  - 双击恢复默认宽度。
- 右侧预览区：
  - 顶部工具栏：缩放、适合宽度、适合页面、页数。
  - 内容区：A4 页面居中显示。

状态与反馈：

- 加载中：右侧显示 A4 骨架页，左侧显示 AI 引导消息骨架。
- 空状态：A4 页面展示空白模块标题，聊天区引导用户输入经历。
- 出错：顶部栏显示「保存失败」，聊天区保留用户输入，提供「重试」。
- 成功：顶部栏显示「已保存」，toast 显示「修改已应用」。

响应式：

- 电脑：左右分栏。
- 平板：左侧聊天栏可收起，右侧预览占满。
- 手机：默认显示聊天，预览通过底部切换按钮打开。

### 6.2 组件层级：左侧聊天栏

结构顺序：

1. 简历上下文区
   - 简历标题
   - 目标岗位输入
   - 保存状态
2. 消息流
   - AI 引导消息
   - 用户消息
   - AI 回复
   - 改动摘要卡片
   - 系统提示
3. 快捷指令
4. 输入区

交互规则：

- 用户点击快捷指令后，指令填入输入框并可编辑。
- 用户发送空内容时，输入框显示错误：「请输入修改要求」。
- AI 处理中，允许用户继续查看历史消息和预览。
- AI 处理中再次发送时，新消息进入队列，消息显示「等待处理」。
- 改动摘要应用完成后，聊天滚动到最新消息。

### 6.3 组件层级：右侧 A4 预览区

结构顺序：

1. 预览工具栏
   - 缩放下拉：50%、75%、100%、适合宽度、适合页面。
   - 页数：`1 / 1`、`1 / 2`。
   - 刷新状态：`预览已更新`。
2. 画布
   - A4 页面列表。
   - 页面阴影。
   - 多页间距。
3. 页面提示
   - 溢出提示。
   - 导出前检查提示。

交互规则：

- 鼠标滚轮滚动画布。
- `Cmd/Ctrl + +` 放大，`Cmd/Ctrl + -` 缩小。
- AI 应用修改时，A4 页面显示半透明刷新层，内容完成后一次性切换。
- 用户预览历史版本时，右侧顶部显示「正在预览历史版本」，并提供「回到当前版本」。

### 6.4 抽屉：版本历史

- 目的：查看所有修改版本，恢复指定版本。
- 入口：顶部栏「历史」。
- 主操作：「恢复此版本」。

页面骨架：

- 抽屉头部：标题「版本历史」、关闭按钮。
- 说明区：显示当前版本时间。
- 列表区：版本项按时间倒序。
- 底部：当前预览版本的操作区。

状态：

- 加载中：显示 5 条版本骨架。
- 空状态：显示「还没有历史版本」，引导用户先应用一次 AI 修改。
- 出错：显示「版本加载失败」，提供「重试」。
- 恢复成功：关闭抽屉，toast 显示「已恢复到所选版本」。

交互规则：

- 点击版本项只预览。
- 点击恢复进入确认弹窗。
- 恢复后生成新的「恢复版本」记录。

### 6.5 弹窗：导出确认

- 目的：确认导出格式并反馈下载结果。
- 入口：「导出 HTML」「导出 PDF」按钮。
- 主操作：「开始导出」。

内容：

- 标题：导出 HTML / 导出 PDF。
- 文件名输入：默认 `我的简历-2026-05-19`。
- 格式说明：HTML 包含完整样式，PDF 使用 A4 页面。
- 页面信息：当前页数、缩放与导出无关的提示。
- 操作区：「取消」「开始导出」。

状态：

- 加载中：主按钮显示「导出中」。
- 成功：自动关闭弹窗，toast 显示「文件已下载」。
- 失败：弹窗内显示原因，保留「重试导出」。

### 6.6 弹窗：恢复确认

- 目的：让用户确认恢复历史版本。
- 标题：「恢复这个版本」
- 内容：显示版本时间、用户指令摘要和影响提示。
- 操作：「取消」「恢复此版本」。
- 成功：关闭弹窗和抽屉，刷新右侧预览。

## 7. 文案与语气

- 整体语气：直接、清楚、稳定，减少解释性长句。
- AI 引导语：「告诉我你的经历、目标岗位，或直接说你想怎么改。」
- 空白简历提示：「从一句话开始，例如：我想做前端开发，有两个项目经历。」
- 保存中：「保存中」
- 保存成功：「已保存」
- 保存失败：「保存失败，点击重试」
- AI 处理中：「正在生成修改计划」
- 应用中：「正在应用修改」
- 预览刷新：「预览已更新」

按钮文案：

- 「发送」
- 「应用修改」
- 「重新生成」
- 「历史」
- 「恢复此版本」
- 「导出 HTML」
- 「导出 PDF」
- 「开始导出」
- 「重试」
- 「关闭」

## 8. 无障碍与可用性

- 所有按钮、输入框、抽屉和弹窗都支持键盘 Tab 聚焦。
- 弹窗打开后焦点进入弹窗，关闭后焦点回到触发按钮。
- 主按钮支持 Enter 触发。
- 聊天输入支持 Shift+Enter 换行。
- 所有图标按钮必须提供 `aria-label`。
- 焦点环使用品牌蓝，宽度 2px。
- 正文与背景对比度达到可读标准。
- 手机可点击区域高度至少 44px。
